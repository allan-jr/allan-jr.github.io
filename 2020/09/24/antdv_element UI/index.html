antdv与element UI组件与API的区别（部分）
2020-09-24 | Allan | git,blog | blog, git | 被浏览次

摘要

本篇主要讲解antdv与element UI组件与API的区别（部分）

分页

1.element 与 ant UI(表格)

elementUI 组件基本都以 el-xxx 开头而 antUI 基本都以 a-xxx 开头，table 中 element 绑定数据的 api 是:data,而 ant 是:data-source。

el-table-column 中以 label 为表头名称，a-table-column 以 title 作为表头名称,api width 区别 ant 中使用必须附带计算单位否则无效。

template 在 elementUI 还是 ant 中都为 template，区别在于 api slot-cope 当在 element 中使用是=”scope”,并以来代表这一行对应表头的内容而 ant 中使用是=”text,record”,并以来代表这一行对应表头的内容。

还有最后一点要提 ant 中 Table 组件是自带分页属性的可使用:pagination=”false”设置去掉默认分页

2.element 与 ant UI(分页)

上述其实已经可以看到 api 有这很明确的不同点了，小编就不多解释了。

这里主要提两点，第一要提的是 ant 分页与 element 分页展示多少条/页，与跳转至某一页的区别,ant 必须要通过 api 设置:showSizeChanger=”true” :showQuickJumper=”true”才会在界面上展示出来，而 elementUI 自带默认展示。

第二要提的点是两者方法的回调参数

3.element 与 ant UI(下拉框)

看的见的区别小编就不多啰嗦了，这里区别主要讲要一点placeholder(占位符)，当两种UI组件都绑定了默认值且为空的情况下，ant中placeholder(占位符)会失效，解决方法取消绑定默认值可以通过@change方法来设置绑定值，如果默认值不为空的建议直接绑定使用就好。

作者心里话

有幸接到一个elementUI+Vue的项目需要迁移几个页面到新项目的任务，该项目使用antdvUI+Vue,原本以为挺简单的直接拿过来用就好了，结果发现还是有很多不同点，很多东西拿到新项目中都失效了。然后仔仔细细的看了一下elementUI与antUI的组件与API调用，发现这里学问还挺大的，以防以后在遇到类似的迁移，先提前简单明了的把大致所要用到的组件及API标指出来，方便以后回顾！
